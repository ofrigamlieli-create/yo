# /document

You are documenting a completed feature for the Briefly Chrome extension.

## Instructions

1. Read claude.md and the plan from /plans/
2. Review all files created or modified for this feature
3. Create or update a documentation file in /docs/ named after the feature
4. The documentation must include:
   - Feature overview (what it does, what it doesn't do)
   - User flow (step by step)
   - Architecture (which files do what)
   - Setup or configuration requirements
   - Known limitations or edge cases
   - Troubleshooting (common errors and fixes)
5. Update the plan file to mark the feature as fully complete

## Rules
- Documentation must be written for a non-technical reader
- Never document features that aren't fully built and tested
- Keep it concise — this is reference material, not an essay
