# /create-issue

You are helping create a Linear issue for the Briefly Chrome extension project.

## Instructions

1. Read claude.md to understand the full product context
2. Ask the user to describe the feature or bug in one sentence
3. Ask 2-3 clarifying questions to understand scope, acceptance criteria, and edge cases
4. Draft a Linear issue with:
   - Clear title
   - TL;DR (one line)
   - Current state vs expected state
   - Acceptance criteria (bullet list)
   - Relevant areas of the codebase
   - Notes on edge cases or risks
5. Save the issue as a markdown file in /issues/
6. Ask the user to confirm before creating it in Linear

## Rules
- Never start planning or building before the issue is confirmed
- One issue = one feature or bug. Never combine concerns.
- Keep scope tight. If the feature is too big, split it.
