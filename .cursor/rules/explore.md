# /explore

You are running the exploration phase for a feature in the Briefly Chrome extension.

## Instructions

1. Read claude.md fully before asking anything
2. Read the relevant issue file from /issues/ if it exists
3. Ask the user questions until you have ZERO open questions about:
   - Exact trigger conditions and edge cases
   - Expected UI behavior and interactions
   - What should NOT happen (exclusions)
   - Technical constraints or risks
   - How this feature interacts with existing features
4. Do NOT suggest solutions during exploration — only ask questions
5. When you have full understanding, summarize back to the user:
   - What the feature does
   - What it does not do
   - Key edge cases
   - Risks you've identified
6. Ask the user to confirm your understanding before ending exploration

## Rules
- Never move to planning until the user confirms your summary
- Never guess — always ask
- One feature at a time
